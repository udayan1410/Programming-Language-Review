1)If - The "if" statement evaluates the test expression inside the parenthesis.

  • If the test expression is evaluated to true(1), statements inside the body of "if" is executed.
  • If the test expression is evaluated to false(0), statements inside the body of "if" are skipped from execution.
Syntax:
if(testExpression)

Example:

int main()
{
int a=5;
	if(a==5)
	{
	cout<<"Hello World !!!";
	}
	return 0;
}
Output: Hello World !!!
In case, a is not equal to 5 i.e the "if" condition is false then the statements inside the body of "if" will be skipped i.e "Hello World !!!" will not be displayed.

Did you know? - If you put semi-colon(;) after if(condition) then it will always return true and the immediate next statement after "if" is executed.


2)if-else - if-else is used for exception handling. When either of the outcomes (true or false) occur, a particular block of code is to be executed.
When "if" condition is false, "else" block is executed and when "if" condition is true, "else" block is skipped.

Syntax:
if(testExpression) {
//statements inside the body of "if"
}
else
{
//statements inside the body of "else"
}
There are 2 types of if-else statements:
  • if...else Ladder - When multiple conditions are to be evaluated if-else ladder is used.

    • Syntax:
    if(testExpression1)
    {
    //statement(s)
    }
    else if(testExpression2)
    {
    //statement(s)
    }
    .
    .
    else
    {
    //statement(s)
    }
    Example:
    
int main()
    
{
    
int number1, number2;
    
cout<<"Enter two integers: ";
    
cscanf("%d%d",&number1,&number2);
    
if(number1==number2)
    
{
cout<<"Result: numbers same"
    
}
else if(number1>number2)
    
{
cout<<"Result: "<<number1;
    
}
else
{
cout<<"Result: "<<number2
    
}
return 0;
}

Result: 25

3)Switch-case - The if..else..if ladder allows you to execute a block code among many alternatives. If you are checking on the value of a single variable in if...else...if, it is better to use switch statement.
The switch statement is often faster than nested if...else (not always). Also, the syntax of switch statement is cleaner and easy to understand.

Syntax:
switch (n)
​{
case constant1: // code to be executed if n is equal to constant1;
break;
case constant2: // code to be executed if n is equal to constant2;
break;
. . .
default: // code to be executed if n doesn't match any constant
}

IMP Note: When a case constant is found that matches the switch expression, control of the program passes to the block of code associated with that case.
Suppose, the value of n is equal to constant2. The compiler executes the statements after case constant2: until break is encountered. When break statement is encountered, switch statement terminates.

Example:

int main()

{
int x=3;

switch(x)

{
	case 1: cout<<"Hi\n";
	break;
	case 2: cout<<"Hello!\n";
	break;
	case 3: cout<<"Welcome\n";
	case 4: cout<<"Welcome to Pune\n";
	break;
default: cout<<"All the best\n";
}
return 0;

}

Output:

Welcome

Welcome to Pune
Here, since there is no "break" statement after case 3, case 4 is also executed and switch statement is terminated as there is "break" after case 4.

IMP Note: When a case constant does NOT match any of the switch expressions then "default" case is executed (provided that it is given in the "switch" body).
